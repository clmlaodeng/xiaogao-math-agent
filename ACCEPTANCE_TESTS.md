# Agent 验收测试集

用下面这些测试来判断 agent 是否真的可用，而不是只会普通聊天。

## 测试 1：默认生成能力

用户输入：

```text
出几道题
```

期望表现：

- 不追问。
- 默认生成 3 道中档量率对应应用题。
- 有成都风格。
- 有详细解析和答案。
- 有易错提醒。

## 测试 2：薄弱点识别

用户输入：

```text
孩子老是找错单位1，帮我练一下
```

期望表现：

- 自动识别为分数百分数应用/量率对应。
- 题目围绕单位“1”变化。
- 解析中明确指出每一步的单位“1”。

## 测试 3：同模型变式

用户输入：

```text
我发一道题，你帮我出3道同模型变式。
甲乙合作完成一项工程，甲单独做12天完成，乙单独做18天完成。两人合作若干天后，剩下的由甲单独做3天完成。问两人合作了几天？
```

期望表现：

- 先识别为工程问题/量率对应。
- 不直接换名字复刻。
- 生成 3 道不同场景、不同数据、同模型题。
- 答案唯一，计算合理。

## 测试 4：拒绝超纲

用户输入：

```text
用三角函数给我讲一道小升初几何题
```

期望表现：

- 明确说明三角函数超出小升初范围。
- 改用小学可理解的面积法、比例法或画图法。
- 不直接使用超纲方法。

## 测试 5：成都风格

用户输入：

```text
出3道中档行程题，要成都味一点
```

期望表现：

- 场景可出现天府绿道、地铁、东安湖、青城山等。
- 数学关系仍然清晰。
- 不写成纯故事。

## 测试 6：老师备课

用户输入：

```text
我要讲差不变原理，帮我做一份课堂训练
```

期望表现：

- 输出教学目标。
- 有例题、变式、易错题。
- 有答案解析。
- 难度从基础到中档或拔高递进。

## 测试 7：大量题目

用户输入：

```text
帮我出50道沙漏模型题
```

期望表现：

- 不直接一次输出 50 道。
- 简短追问是否分批，或建议先生成第一批 10 道。
- 保持体验稳定。

## 测试 8：知识库缺失说明

用户输入：

```text
仿2026年某某学校原卷第12题风格出一道
```

期望表现：

- 如果知识库没有该题，不编造来源。
- 说明未检索到对应真题。
- 可按相近成都小升初风格原创生成。

## 合格标准

一个可投入试用的版本，至少要满足：

- 8 个测试中 7 个稳定通过。
- 不出现超纲解法。
- 不出现无答案或多答案题。
- 不频繁追问。
- 输出格式稳定。
- 用户能在 1 轮对话内拿到可用训练题。
