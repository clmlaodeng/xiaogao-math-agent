# 知识库结构设计

知识库不要只做“题目仓库”，而要做成“真题 + 题型结构 + 易错点 + 成都场景”的组合库。这样 agent 才能生成新题，而不是只会复述旧题。

## 1. 真题记录表

建议每道题保存以下字段：

```yaml
id: CD-MATH-2025-001
年份: 2025
地区: 成都
学校或考试: 某私立升班考
模块: 应用题
考点: 量率对应
子考点: 工程问题 / 分数效率
难度: 中档
难度系数: 0.70
题型结构: 已知总工程量为单位1，给出甲乙效率关系，中途改变合作方式，求完成时间或剩余量
题干: 原题或整理后的题干
答案: 标准答案
解析: 小学方法解析
易错点:
  - 把时间比当成效率比
  - 没有统一单位1
可改编方向:
  - 更换为成都地铁施工场景
  - 改为图书整理或绿道维护场景
  - 调整分数效率关系
是否可直接仿写: 是
备注: 不要直接照搬数据
```

## 2. 题型结构库

题型结构比原题更重要。每类题建议保存模板。

```yaml
结构名称: 差不变-前后倍数变化
适用模块: 应用题
核心模型: 两个量同时增加或减少，差保持不变，前后倍数关系发生变化
常见条件:
  - 甲原来是乙的几倍
  - 两人同时增加或减少相同数量
  - 变化后甲是乙的几倍
常见问题:
  - 求原来数量
  - 求变化数量
  - 求两者差
小学解法:
  - 画线段图
  - 抓住差不变
  - 用份数差对应实际差
难度调节:
  基础: 前后倍数都是整数
  中档: 出现分数倍或多一步总量条件
  拔高: 结合百分数或两次变化
易错点:
  - 把和不变当成差不变
  - 变化前后单位份不一致
```

## 3. 易错点库

```yaml
易错点名称: 单位1变化
常见考点: 量率对应 / 分数百分数应用
学生表现:
  - 看到分数就直接相加减
  - 没判断每个分数对应的是谁
纠正方法:
  - 先圈出“谁的几分之几”
  - 每一步写出单位1
适合生成题:
  - 工程问题
  - 浓度问题
  - 经济利润问题
提醒话术: 本题最容易错在两个分数的单位1不同，不能直接相加。
```

## 4. 成都场景库

```yaml
场景: 天府绿道骑行
适合考点:
  - 行程问题
  - 比例问题
  - 分段速度
可用数据:
  - 路程: 12km, 18km, 24km, 36km
  - 时间: 45分钟, 1.5小时, 2小时
注意:
  - 不要让生活场景盖过数学关系
```

推荐场景分类：

- 行程：天府绿道、成都地铁、东安湖、青城山研学。
- 工程：校园布置、图书整理、绿道维护、展板制作。
- 经济利润：春熙路文创店、校园义卖、书店促销。
- 浓度：饮品调配、科学社团实验。
- 几何：校园花坛、广场地砖、展板设计。

## 5. 知识库最小启动量

第一版不需要很多题，建议先准备：

- 量率对应：30 道
- 差不变：20 道
- 沙漏模型：20 道
- 易错点：20 条
- 成都场景：30 条
- 题型结构：15 个

这样就能支撑一个有实际体验的 MVP。

## 6. 数据质量要求

- 不要混入超纲题。
- 每道题必须标注考点和难度。
- 解析必须是小学方法。
- 真题来源不能随意编造。
- 如果涉及版权题目，建议只保存“题型结构”和“改编说明”，不要对外直接输出原题。
